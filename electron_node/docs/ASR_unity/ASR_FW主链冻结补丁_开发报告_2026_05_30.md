## 2. 架构变更

### 2.1 业务文本解析（单一 fallback 链）

新增 `resolveBusinessAsrText` / `resolveBusinessAsrTextSource`（`pipeline/post-asr-routing.ts`）：

```text
segmentForJobResult（非空）→ 业务文本
  └─ 空 → asrText（唯一 fallback）
禁止：rawAsrText、repairedText
```

消费方统一：

| 模块 | 函数 |
|------|------|
| NMT | `getTextForTranslation` → `resolveBusinessAsrText` |
| Result | `buildJobResult.text_asr` → `resolveBusinessAsrText` |
| 日志 | `translation-step` → `resolveBusinessAsrTextSource` → `[TRANSLATION_INPUT] source=...` |

**决策 §3.3：** 不新增 `extra.translation_input_source`，以日志 + 单测作为门禁。

### 2.2 写锁与 Session 边界（文档化，逻辑未改）

- `asrRepairApplied`：FW 仅 `decision.approved.length > 0` 时置 true  
- Session `RollingTurn.finalText`：turn 快照，非 JobContext SSOT  
- Intent HTTP wire 键 `repairedText` → 值来自 `finalText`（wire 兼容）

### 2.3 类型清理

- 删除 `JobResult.text_asr_repaired` 遗留字段（`inference-service.ts`）
