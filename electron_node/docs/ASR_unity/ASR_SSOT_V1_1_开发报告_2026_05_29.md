## 1. 开发目标

消除 ASR 主链「双真值源」（`repairedText` vs `segmentForJobResult`），以 **`segmentForJobResult` 为唯一业务 SSOT**，贯穿 FW Detector、Aggregation、Dedup、Semantic Repair、NMT、Result Builder。

---

## 2. 架构变更摘要

### 2.1 字段职责（冻结后）

| 字段 | 职责 | 可写 Step |
|------|------|-----------|
| `rawAsrText` | ASR immutable 原文；Detector 输入 D1；`extra.raw_asr_text` | 仅 `asr-step` 首写 |
| `asrText` | 多 batch 拼接观测 / diagnostics | ASR 相关 |
| `segmentForJobResult` | **唯一业务真值**（NMT / `text_asr` / 聚合输出） | FW apply、Agg、5015、5016（未写锁时） |

### 2.2 已删除 / 废弃

- `JobContext.repairedText`
- `syncRepairedTextBaseline()`
- `extra.text_asr_repaired`
- 主链「repairedText 优先于 segment」分支

### 2.3 写锁

- `isSegmentWriteLocked()` ← `asrRepairApplied === true`
- FW apply 后 5015/5016 不得覆盖 segment

### 2.4 主链数据流

```text
ASR → rawAsrText (freeze) + segmentForJobResult (baseline)
  → FW (detect: rawAsrText, apply → segmentForJobResult)
  → AGGREGATION (read/write segmentForJobResult)
  → Dedup / 5015 / Translation / ResultBuilder
```

**turn non-finalize：** `appendTurnSegment(segment)` + `shouldDeferTranslation=true`，**不清** segment。

## 5. 已知遗留（非阻塞）

1. **Intent HTTP / Python** wire 字段仍名 `repairedText`，Node 映射 `finalText` → 兼容键，非双 SSOT。
2. **Aggregator 局部参数名** `repairedText` 语义已是 segment 文本，可后续重命名。
3. **`clear_code_cache_comprehensive.ps1` 会删 `renderer/dist`**，清理后须 `npm run build` 再启动，否则 UI 空白（与 SSOT 无关）。
