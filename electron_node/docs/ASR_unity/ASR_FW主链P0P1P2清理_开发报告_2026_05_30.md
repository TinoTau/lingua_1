## 1. 开发目标

在 SSOT V1.1 + 冻结补丁基础上，完成三阶段清理：

| 阶段 | 目标 |
|------|------|
| **P0** | 删除 `asrText` / `rawAsrText` fallback；业务文本只读 `segmentForJobResult` |
| **P1** | FW 模式 Result extra 最小化；合并 FINAL 冻结文档 |
| **P2** | Recover 源码迁入 `legacy/recover/`；Intent wire `repairedText` → `finalText` |

**不改主链控制流：** `ASR → FW_SPAN_DETECTOR → AGGREGATION → DEDUP → TRANSLATION`

---

## 2. 架构变更摘要

### 2.1 业务文本（P0）

```ts
// pipeline/post-asr-routing.ts
export function resolveBusinessAsrText(ctx: JobContext): string {
  return (ctx.segmentForJobResult ?? '').trim();
}
```

- 删除 `resolveBusinessAsrTextSource`、`isRecoverWriteLocked` 别名
- 删除 `JobContext.shouldSendToSemanticRepair`
- `aggregation-step`：`currentSegment` 仅读 segment，segment 空 → defer/skip

### 2.2 Result extra（P1）

- FW 引擎：`buildFwResultExtra` — 无 `sentence_repair` / `window_candidates` / `ctc_nbest_preserved`
- Recover 引擎：仍走 `buildRecoverResultExtra`（从 `legacy/recover/recover-contract`）

### 2.3 Recover 物理隔离（P2）

```text
main/src/legacy/recover/
  steps/lexicon-recall-step.ts
  steps/sentence-repair-step.ts
  recover-contract.ts
  v5-metrics.ts
  asr-repair/sentence-expansion/
  asr-repair/sentence-rerank/   （rerank/apply 等，非 KenLM scorer）
```

**KenLM 保留在主链（FW 共用）：**

```text
asr-repair/kenlm-batch-types.ts          ← 新建共享类型
asr-repair/sentence-rerank/kenlm-scorer.ts
asr-repair/kenlm-span-gate.ts            ← FW weak_veto
```

原因：FW span 决策依赖 `createKenlmBatchScorer()`，若整体迁入 legacy 会导致 FW import legacy，违反隔离目标。

### 2.4 Intent wire（P2）

| 位置 | 变更 |
|------|------|
| `lexicon-intent-prompt-builder.ts` | `turns[].finalText` |
| `intent-warmup.ts` | warmup payload `finalText` |
| `services/lexicon_intent_cpu/service.py` | `TurnInput.finalText` |
| aggregator / semantic-repair-validator | 参数与日志字段去 `repairedText` 命名 |
