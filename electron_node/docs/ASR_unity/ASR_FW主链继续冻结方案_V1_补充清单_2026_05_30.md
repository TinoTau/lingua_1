## 3. 方案未写清、必须在冻结方案中补充的约束

### 3.1 Translation 输入 fallback（P0 冻结语义）

**代码事实**（`post-asr-routing.ts` → `getTextForTranslation`）：

```ts
// segment 非空 → segment；否则 → asrText；均空 → ''
const segment = (ctx.segmentForJobResult ?? '').trim();
if (segment.length > 0) return segment;
return (ctx.asrText ?? '').trim();
```

**方案缺口：** §5.4 写「只读 segment + fail-fast」，与实现不符。

**建议冻结约束：**

- [ ] **主路径：** NMT 输入必须来自非空 `segmentForJobResult`  
- [ ] **允许 fallback：** 仅当 segment 为空且 `asrText` 非空时，可 fallback（诊断/异常 job）；须打日志 `source=asrText`  
- [ ] **禁止 fallback：** 不得 fallback 到 `rawAsrText`、不得读取已删除的 `repairedText`  
- [ ] 批测 / manual_cut 路径：契约要求 segment 非空，**不应出现 `source=asrText`**

---

### 3.2 Result Builder fallback（P0）

**代码事实**（`result-builder.ts`）：

```ts
const finalAsrText = (ctx.segmentForJobResult ?? ctx.asrText ?? '').trim();
result.text_asr = finalAsrText;
```

**建议冻结约束：**

- [ ] `text_asr` 优先 `segmentForJobResult`  
- [ ] segment 为空时允许 `asrText` 兜底（与 NMT 同源规则一致）  
- [ ] **`extra.raw_asr_text` 永远来自 `ctx.rawAsrText`**，不得作为 `text_asr`  
- [ ] **`text_asr_repaired` 不得再写入 extra**（`JobResult` 类型仍保留字段定义，属遗留类型，见 §4.3）
