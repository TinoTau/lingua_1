# ASR→FW 主链冻结后清理方案（P0/P1/P2）

版本：V1.0  
日期：2026-05-30  
适用范围：electron-node / ASR→FW Detector→Aggregation→Dedup→Translation 冻结主链  
依据：`ASR_FW主链冻结后只读清理审计_2026_05_30.md`  
原则：不考虑兼容；不改主链控制流；只删除 fallback、冗余别名、旧 Recover 噪音与历史 wire 命名

---

# 1. 背景与目标

当前主链已经冻结：

```text
ASR
→ FW_SPAN_DETECTOR
→ AGGREGATION
→ DEDUP
→ TRANSLATION
```

当前业务文本 SSOT：

```ts
ctx.segmentForJobResult
```

已确认：

- `repairedText` 已退出 Pipeline JobContext 主链
- `segmentForJobResult` 已贯穿 FW / Aggregation / Dedup / NMT / text_asr
- Recover steps 不进入 FW 默认链
- CTC / Sherpa / n-best / rerun / secondary decode 不进入默认主链
- dialog_200 × 50 已通过
- freeze-contract 与 fw-gate 已保护 SSOT

本清理方案的目标不是再次调整架构，而是把冻结后的主链进一步压缩到最简状态：

```text
少 fallback
少别名
少历史 extra
少旧文档
少 wire 混淆
```

最终目标：

```text
主链只保留一种业务文本：
segmentForJobResult
```

---

# 2. 总体原则

## 2.1 不改主链控制流

禁止修改：

```text
ASR → FW_SPAN_DETECTOR → AGGREGATION → DEDUP → TRANSLATION
```

允许修改：

- 字段 fallback
- 冗余函数别名
- tests 旧契约
- Recover extra 输出
- legacy Recover 文件位置
- 文档体系
- wire 命名

---

## 2.2 删除历史兼容

项目未上线，不保留旧行为。

优先删除：

- `asrText` fallback
- `rawAsrText` fallback
- `repairedText` wire 混淆
- Recover extra 默认打包
- 旧文档多版本冲突
- 旧测试保护旧契约

---

## 2.3 保留必要语义

必须保留：

```ts
rawAsrText
```

用途：

```text
immutable ASR 原文
FW Detector 输入
extra.raw_asr_text
审计与批测
```

必须保留：

```ts
asrText
```

但只作为：

```text
diagnostics / debug / ASR 拼接观测
```

禁止作为：

```text
NMT input
text_asr
Aggregation currentSegment
```

必须保留：

```ts
segmentForJobResult
```

用途：

```text
唯一业务文本源
```

# 3. P0：删除 fallback 和冗余别名

## 3.1 P0 目标

P0 是低风险、高收益清理。

目标：

```text
NMT / Result / Aggregation 只读 segmentForJobResult
```

删除：

```text
asrText fallback
rawAsrText fallback
isRecoverWriteLocked 别名
shouldSendToSemanticRepair 冗余字段
```

---

## 3.2 P0 改造前

### Translation

```ts
segmentForJobResult
  → asrText
  → ''
```

### Result Builder

```ts
segmentForJobResult
  → asrText
  → ''
```

### Aggregation currentSegment

```ts
segmentForJobResult
  → rawAsrText(FW)
  → asrText
```

问题：

```text
fallback 会掩盖 segmentForJobResult 未初始化的问题
```

---

## 3.3 P0 改造后

### Translation

```ts
const text = ctx.segmentForJobResult?.trim() ?? "";
```

如果为空：

```text
skip Translation
或 fail-fast diagnostics
```

禁止 fallback 到：

```ts
ctx.asrText
ctx.rawAsrText
ctx.repairedText
```

---

### Result Builder

```ts
text_asr = ctx.segmentForJobResult?.trim() ?? "";
```

禁止 fallback 到：

```ts
asrText
rawAsrText
repairedText
```

---

### Aggregation

```ts
currentSegment = ctx.segmentForJobResult?.trim() ?? "";
```

如果为空：

```text
segmentReady=false
defer / skip
```

## 5.4 P2 wire repairedText 问题

当前存在非 Pipeline 主链的 `repairedText`：

```text
Intent HTTP wire
warmup payload
validator 局部变量
旧 mock
```

其中：

```text
wire repairedText = Session finalText
```

不是 Pipeline JobContext 字段。

但是命名容易误导。

---

## 5.5 P2 wire 改造选项

### 选项 A：同步改名

将 wire：

```json
{
  "repairedText": "..."
}
```

改为：

```json
{
  "finalText": "..."
}
```

需要同步：

- Node
- Python `lexicon_intent_cpu`
- prompt builder
- warmup
- tests

优点：

```text
彻底清理 repairedText 命名
```

缺点：

```text
跨服务改动
```

---

### 选项 B：保留 wire 名，明确隔离

保留：

```json
{
  "repairedText": "..."
}
```

但在类型与文档中标注：

```text
wire legacy key only
value = finalText
not Pipeline SSOT
```

优点：

```text
风险低
```

缺点：

```text
repairedText 字符串仍存在
```

---

## 5.6 推荐

P2 推荐分两步：

```text
P2-A：
Recover 代码移动到 legacy/recover

P2-B：
wire repairedText 改为 finalText
```

如果 Python 同步成本较高：

```text
先做 B 隔离文档
再做 A 改名
```

# 8. 推荐实施顺序

```text
第一轮：P0
目标：删除 fallback，压实 SSOT。

第二轮：P1
目标：减少 Recover extra，合并 FINAL 文档。

第三轮：P2-A
```

```text

P2 是目录和跨服务清理
```

分开做更容易回归。
