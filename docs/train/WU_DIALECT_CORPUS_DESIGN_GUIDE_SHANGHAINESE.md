# 1. 概述

吴语系方言（尤其上海话）具有：

* 发音高度多变（同词多发音，跨区差异显著）
* 情绪/语气表达丰富
* 大量普通话没有对应词汇
* 无统一文字化体系
* 同音不同义与不同音同义现象极多

因此常规 ASR 采集（词+短句）无法提供足够训练语义/语用能力。
本规范定义一个三层体系：

> **单词类（音系覆盖） + 短句类（句法/连读） + 情景类（语用/情绪/上下文）**
> 用普通话作为统一标签（Label），实现
> 「方言语音 → 普通话文本 → 多语言翻译」链路。

---

# 2. 数据采集总体原则

1. **统一语义标签（普通话文本）**

   * 方言音频对应普通话句子（作为语义锚点）。
   * 不依赖不稳定的“上海话汉字混排”。

2. **覆盖所有常见发音变体**

   * “明天”= ming zoq / meng zoq
   * “这里”= zeli / zili / dji li
   * 等等。

3. **覆盖音系差异与语调差异**

   * 不同年龄、性别、区县口音。
   * 情绪、语速、环境噪声。

4. **覆盖自然对话的语用现象**

   * 生气
   * 委屈
   * 调侃
   * 催促
   * 闲聊
   * 拒绝
   * 等等。

5. **训练标签永远使用普通话**

   * 方便作为后续 Qwen/M2M100 输入。
   * 避免方言汉字表记不准确导致训练混乱。

# 3. 三层语料体系设计

## 3.1 第一层：单词类（词 / 音节级）

### 目标

* 覆盖吴语“核心音系变体”。
* 专门针对“多发音词”“难发音点”“最常见的音变”。

### 示例类别

* 时间类：今天 / 明天 / 昨天 / 现在 / 刚刚
* 高频动词：吃、喝、走、拿、放、叫、看
* 人称代词：侬、阿拉、伊、侬拉
* 疑问词：啥个、哪能、啥地方、啥辰光
* 构词变体：有、无、弄、勿、伐、勒

### 数据量建议

占总语料 **15–25%**

---

## 3.2 第二层：短句类（2–6 字 / 词结构）

### 目标

* 训练句法、连读、变调、弱化。
* 比单词类更接近自然语流。
* 主要承担 Whisper-small 的自适应训练。

### 示例模板

* 我明天要去公司。
* 今朝天冷。
* 侬刚刚去了哪里？
* 弄拿到伐？
* 阿拉勒路上。

### 数据量建议

占总语料 **40–50%**

---

## 3.3 第三层：情景类（对话 / 情绪 / 场景）

### 目标

解决方言最核心的语义问题：

> **上海话中的情绪词、语气词、态度表达、语用单位，普通话没有一一对应，
> 必须依靠场景才能稳定定义含义。**

### 典型情绪类别

* 委屈
* 生气
* 开心
* 无奈
* 嫌弃
* 调侃
* 恐惧
* 疑惑

### 典型场景类别

* 买菜
* 地铁
* 打车
* 公司会议
* 家庭对话
* 争吵
* 打电话
* 求助
* 点菜

### 示例（含普通话标签）

| 上海话语音（示例，仅作为说明）             | 普通话标签            | 场景    |
| --------------------------- | ---------------- | ----- |
| “侬勿要来西搞，阿拉今朝心头 ka la la 的。” | “你别来惹我，我今天心情不好。” | 情绪：烦躁 |
| “今朝路上堵脱噶，我迟来一眼眼。”           | “路上堵车了，我会晚到一点。”  | 上下班   |
| “侬再讲阿拉要发起来了喔。”              | “你再说我就要生气了。”     | 争吵    |

### 数据量建议

占总语料 **30–40%**

# 4. 标签（Label）规范：为何统一普通话

原因：

1. 上海话汉字体系不统一
2. 拼音体系不稳定（IPA/Wu拼/自造体系）
3. 普通话作为“统一语义写法”可稳定接入 NLP
4. 方便后续用 Qwen/M2M100 自动翻译

采集到的语音为：

> **吴语系方言语音**
> 标签为：

> **普通话句子（简体）**

```json
  "task_type": "word | short_sentence | scenario",
  "scenario_type": "optional string",
  "emotion": "optional string",
  "metadata": {
    "gender": "M/F",
    "age": "integer",
    "region": "string",
    "noise_level": "low/medium/high"
  }
}
```

# 6. Web 端采集任务设计

### 核心设计

* 主行显示 **普通话句子**
* 辅助说明：

  > “请用上海话读出上面的句子。”
* 可选：示例音频（方言），但不作为标签来源
* 任务类型包含：

  * 单词
  * 短句
  * 情景（对话、情绪）

### 任务示例

```
【文本】我今天有点心烦。
【说明】请用上海话读出上面的句子。
【标签】"我今天有点心烦。"
```

# 7. 采集覆盖策略（多变体 / 说话人）

### 建议覆盖

* 男 + 女
* 老中青
* 市区（黄浦/徐汇/静安）
* 郊区（嘉定/青浦/奉贤）
* 快速 / 正常 / 慢速
* 情绪加强版
* 自然对话版

### 多变体词覆盖

例：“明天”

* ming zoq
* meng zoq
* ming zaq
* meng zaq

让模型学习“发音等价类”。

---

# 8. 标注流程

1. 系统生成普通话句子（统一写法）
2. 用户读上海话语音
3. 标签（text_label_mandarin）固定为普通话
4. 数据上传到训练集

训练本质是：

> **吴语语音（多变体） → 普通话统一文本**

---

# 9. 数据清洗与质量标准

* 去除背景噪声 > 90dB
* 去除录歪的问题句
* 保留轻微噪声、连读、自然停顿（有利于训练）
* 针对情绪类：确保情绪准确
* 核查文本标签是否与语义一致

---

# 10. 训练建议

### 推荐训练路线

Whisper-small（微调）
→ LoRA
→ 多节点异步
→ 每日增量训练
→ Distillation from Whisper-large
→ Qwen 翻译普通话到英文

### 建议训练数据量

| 阶段  | 数量        | 目标        |
| --- | --------- | --------- |
| MVP | 5000 句    | 基础吴语识别    |
| V1  | 20,000 句  | 情绪 & 场景识别 |
| V2  | 50,000+ 句 | 稳定的 ST 管线 |

---

# 11. 测试集与评估指标

### ASR/ST 评估

* CER（汉字错误率）
* SER（语义错误率）：是否翻译成正确普通话
* 情绪分类准确度（可选）

### 多语言翻译评估

* BLEU
* METEOR
* Human Rating（人工评分）

建议测试集占总数据的 10%。
